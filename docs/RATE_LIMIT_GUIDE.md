# Rate Limit 가이드

## 문제: "429 Too Many Requests" 에러

Yahoo Finance API는 요청 빈도에 제한이 있습니다. 너무 많은 요청을 보내면 일시적으로 차단됩니다.

## 해결 방법

### 1. 자동 재시도 (이미 구현됨)

코드에 자동 재시도 로직이 포함되어 있습니다:
- 최대 3번 재시도
- 점진적 대기 시간 (2초, 4초, 6초)

### 2. 요청 간 딜레이

각 요청 사이에 0.1초 딜레이가 자동으로 추가됩니다.

### 3. 사용자 액션

**에러가 발생하면:**
1. 잠시 기다린 후 다시 시도 (1-2분)
2. 다른 종목으로 테스트
3. 서버를 재시작하지 마세요 (재시작해도 같은 문제)

### 4. 개발 중 팁

**테스트할 때:**
- 한 번에 여러 종목을 연속으로 검색하지 마세요
- 각 검색 사이에 몇 초 간격을 두세요
- 브라우저를 새로고침하지 마세요 (캐시 활용)

## 에러 메시지

### "429 Too Many Requests"
→ 잠시 기다린 후 다시 시도하세요

### "No data found for symbol"
→ 심볼 형식 확인:
- 주식: `AAPL`, `TSLA` (심볼만)
- 암호화폐: `BTC-USD`, `ETH-USD` (반드시 -USD)

### "Invalid symbol"
→ 올바른 심볼인지 확인하세요

## 대안 (향후 구현 가능)

1. **캐싱**: 같은 종목 데이터를 일정 시간 캐시
2. **배치 요청**: 여러 종목을 한 번에 요청
3. **대체 API**: 다른 데이터 소스 사용

## 현재 상태

- ✅ 자동 재시도 구현됨
- ✅ 에러 메시지 개선됨
- ⏳ 캐싱 (향후 구현)
